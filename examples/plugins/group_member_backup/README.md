# 群成员备份恢复插件

## 功能概述

该插件提供QQ群成员的备份和恢复功能，支持：
- 导出群成员信息到JSON文件
- 从JSON文件恢复群成员（发送加群邀请）
- 自定义邀请间隔时间，避免触发风控
- 后台任务处理，不阻塞主事件循环
- 实时查询恢复任务状态

## 命令列表

### 1. `/backup`
备份当前群的所有成员信息

**使用示例：**
```
/backup
```

**功能：**
- 获取群内所有成员信息
- 生成JSON格式的备份文件
- 将文件发送到群中
- 显示备份统计信息

**备份文件格式：**
```json
{
  "group_id": "123456789",
  "backup_time": 1704528000000,
  "member_count": 500,
  "members": [
    {
      "user_id": 111111,
      "nickname": "用户昵称",
      "card": "群名片",
      "role": "member",
      "title": "头衔"
    }
  ]
}
```

### 2. `/restore [min,max]`
恢复群成员（需要回复一个备份JSON文件）

**使用示例：**
```
/restore              # 使用默认间隔 5-10秒
/restore 60,120      # 使用自定义间隔 60-120秒
```

**参数说明：**
- `min`: 最小间隔时间（秒）
- `max`: 最大间隔时间（秒）
- 默认值：5,10（即每次邀请间隔5-10秒随机）

**功能：**
- 解析备份文件
- 在后台任务中逐个发送加群邀请
- 随机间隔时间，避免触发风控
- 记录成功和失败的邀请
- 完成后发送统计消息

**使用步骤：**
1. 回复一条包含备份JSON文件的消息
2. 在回复中输入 `/restore [min,max]` 命令

### 3. `/restore_status`
查询当前恢复任务的状态

**使用示例：**
```
/restore_status
```

**显示信息：**
- 任务运行状态
- 总成员数
- 待处理数量
- 成功数量
- 失败数量
- 已运行时间

## 配置说明

在 `config.toml` 中添加以下配置：

```toml
[plugins.group_member_backup]
enabled = true
callbacks = ["on_message"]

[plugins.group_member_backup.config]
# 允许使用此功能的群列表（留空表示允许所有群）
allowed_groups = [
    "123456789",
    "987654321"
]

# 允许使用此功能的管理员用户QQ号（留空表示允许所有用户）
admin_users = [
    "111111111",
    "222222222"
]
```

### 配置项说明

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `enabled` | boolean | 是否启用插件 | true |
| `allowed_groups` | array | 允许使用的群列表 | [] (所有群) |
| `admin_users` | array | 允许使用的管理员用户 | [] (所有用户) |

## 权限控制

- 只有配置的管理员用户可以使用备份和恢复功能
- 只能在配置的群中使用该功能
- 如果 `allowed_groups` 或 `admin_users` 为空，则不做限制

## 安全建议

1. **限制管理员用户**：建议在 `admin_users` 中只添加可信任的管理员QQ号
2. **合理设置间隔时间**：建议邀请间隔不少于30秒，避免触发腾讯风控
3. **分批恢复**：如果成员数量很多，建议分多次恢复
4. **定期备份**：建议定期备份群成员信息

## 注意事项

### 关于恢复功能的说明

由于OneBot11协议的限制，当前实现无法直接发送群邀请。实际的邀请逻辑需要配合以下方式：

1. **手动邀请**：Bot主动向用户发送好友请求，然后邀请入群
2. **扫码邀请**：生成群邀请二维码，让用户扫码入群
3. **配合QQ客户端**：使用QQ客户端的群管理功能批量邀请

**当前实现的处理方式：**
- 记录需要恢复的用户列表
- 按照指定间隔处理每个用户
- 记录处理结果（成功/失败）
- 发送统计报告

**建议扩展方向：**
- 集成第三方邀请API
- 配合Bot的好友系统自动发送邀请
- 生成邀请链接或二维码

### 风控说明

频繁的群操作可能触发腾讯风控，导致：
- 账号被限制功能
- 无法发送消息
- 账号被封禁

**建议：**
- 使用较长的间隔时间（60秒以上）
- 分批次恢复成员
- 避免在短时间内大量操作

## 编译安装

1. 确保已安装依赖：
   - Boost (asio, system, thread)
   - nlohmann_json

2. 在主项目CMakeLists.txt中已自动包含

3. 编译：
```bash
mkdir build && cd build
cmake ..
make group_member_backup
```

4. 插件将输出到 `build/plugins/libgroup_member_backup.so`（或 `.dll`/`.dylib`）

## 故障排除

### 备份失败
- 检查Bot是否有获取群成员列表的权限
- 确认Bot在该群中
- 查看日志文件获取详细错误信息

### 恢复失败
- 确认备份文件格式正确
- 检查是否有写入临时文件的权限
- 确认网络连接正常

### 任务不执行
- 检查是否在 `allowed_groups` 和 `admin_users` 白名单中
- 确认插件已正确加载
- 查看日志确认是否有错误

### 核心类说明

- `GroupMemberBackupPlugin`: 主插件类

### 关键方法

- `handle_backup_command()`: 处理备份命令

- `execute_restore_task()`: 在后台线程执行恢复任务
- `handle_status_command()`: 查询任务状态
